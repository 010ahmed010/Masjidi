

Masjid Project

Rules:
    1. Use MERN stack technologies.
    2. Use Tailwind CSS for styling.
    3. This is an Arabic project only, so use a modern Arabic style and appropriate Google fonts.
    4. Use Font Awesome for icons.
    5. Mirror Description Method (MDD): each file represents a page or a feature description.
    6. Organize frontend under a `client` directory, backend under `server`, and reusable UI components under `components`.

Main Context Masjidi Project:
    Masjid project for teaching students Quran and Islamic sciences. It consists of three sides: Public Side, Admin Side, and Teacher Side.
    Public Side: The Home page contains reusable header/footer, hero section, announcements, honor section, current occasions, course listings, contact page, and attendance summaries. These public sections rely on admin-managed content and teacher-provided attendance data.
    Admin Side: Admin controls Honor and Occasions sections, manages teacher accounts, assigns students to teachers, adds students to the school, issues certificates, manages news and contact info, and monitors overall school activity.
    Teacher Side: Teachers manage their assigned classes, set weekly lesson plans, take attendance, and nominate students for the honor wall. Teacher actions feed the public HomePage and are reviewed by admin where needed.
